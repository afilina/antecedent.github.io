Patchwork

Setup

Technically, the process of integrating Patchwork into your code consists only of inserting a require statement somewhere in your code:

require "Patchwork.phar";

However, it matters greatly where exactly this statement is placed.

This is because Patchwork preprocesses PHP code, and only those functions and methods that come from preprocessed files will be redefinable. A rule for finding out which files will be preprocessed and which won't could be formulated as follows:

  • The entry script will not be preprocessed.
  • The file containing the statement that imports Patchwork will not be preprocessed.
  • All files imported (using an include/require statement) earlier than Patchwork will not be preprocessed.
  • All files imported after Patchwork will be preprocessed.

The following graphic, where preprocessed files are colored green and non-preprocessed ones red, should help you understand this rule. The only functions that would be redefinable in such a setup would be eggs() and bacon().

And that's it. By now you should already be able to start redefining functions, and the Usage document is here to help you with that.

If you, however, are having problems with importing Patchwork, please first check if your PHP installation meets the requirements. If the problem turns out to be unrelated, please report it on GitHub. Thank you!
